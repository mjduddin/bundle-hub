5E/4F Bundle Hub Administrator Notes • Data Access

Where the data lives (and why it is not on this site)

By design, this public app never reads data back. It only inserts anonymous rows. All reading happens in the Supabase console, which requires the departmental admin login.

Reading analytics and survey data

  1. Sign in to Supabase

    Use the shared departmental account (see the README in the GitHub repository). Open the project, then Table Editor.

  2. Review the two tables

    events holds landing views, bundle views, dwell times, and feedback clicks. survey_responses holds quick feedback and baseline/post surveys.

  3. Export for run charts

    Use the CSV export button (or the SQL editor) and build weekly run charts in Excel for the QI/LEAN reviews. Report only in aggregate.

Security model — verify before every audit

Row-Level Security (RLS) must be enabled on both tables with INSERT-only policies for the anonymous role and no public SELECT. The public anon key shipped in this site is only safe under that configuration. The exact SQL is in the README.

Updating content

Edit bundles.js in the GitHub repository, commit, and push — Netlify redeploys automatically. No QR codes need reprinting. Set "draft": true on any bundle that has not cleared the approval process; drafts are hidden unless a reviewer adds ?preview=1 to the URL.
